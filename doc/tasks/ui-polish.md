# UI 打磨 — 最小可执行任务

> 对应里程碑：M6
> 依赖：所有功能模块完成
> 完成后验收：响应式布局、美观的 Tailwind 样式、流畅的交互体验

---

## 1. 响应式布局

- [ ] **1.1** 活动卡片网格：
  - 桌面端（≥1024px）：3 列
  - 平板端（≥640px）：2 列
  - 手机端（<640px）：1 列

- [ ] **1.2** 导航栏响应式：
  - 桌面端：完整导航栏（Logo、搜索框、通知、用户菜单）
  - 手机端：汉堡菜单，点击展开垂直菜单

- [ ] **1.3** 表单响应式：
  - 桌面端：表单最大宽度 640px，居中
  - 移动端：全宽，输入框字体不小于 16px（防止 iOS 缩放）

- [ ] **1.4** 报名管理页 Tab 响应式：
  - 桌面端：水平 Tab
  - 移动端：可水平滚动的 Tab

## 2. Tailwind 样式美化

- [ ] **2.1** 统一设计 Token（在 `base.html` 或自定义 CSS 中定义）：
  - 主色、辅色、成功/警告/错误色
  - 圆角、阴影、间距统一

- [ ] **2.2** 活动卡片样式：
  - 圆角卡片 + 阴影 hover 效果
  - 封面图裁剪（`object-cover`），统一高度
  - 分类/标签彩色 Badge
  - 费用：免费显示绿色"免费"，收费显示金额

- [ ] **2.3** 按钮样式：
  - 主按钮（报名、提交、通过）→ 主色
  - 次按钮（取消、编辑）→ 灰色
  - 危险按钮（取消活动、拒绝、删除）→ 红色
  - Disabled 状态 → 灰色 + 不可点击

- [ ] **2.4** 表单样式：
  - 输入框、下拉框、文本域统一样式
  - Focus 状态：主色边框 + 外发光
  - 错误状态：红色边框 + 错误提示文本

- [ ] **2.5** 状态标签：
  - 待审核 → 黄色 Badge
  - 已通过 → 绿色 Badge
  - 已拒绝 → 红色 Badge
  - 候补 → 蓝色 Badge
  - 已取消 → 灰色 Badge
  - 已签到 → 绿色 Badge + ✓

- [ ] **2.6** 空状态占位：
  - 无搜索结果 → "没有找到相关活动，试试其他关键词？"
  - 无通知 → "暂无通知"
  - 无报名记录 → "还没有报名任何活动，去首页看看吧"

## 3. 交互优化

- [ ] **3.1** Flash 消息：
  - 成功消息：绿色横幅，3 秒自动消失
  - 错误消息：红色横幅，需手动关闭（或 5 秒自动消失）
  - 消息从页面顶部滑入

- [ ] **3.2** 倒计时实时更新：
  - 所有活动卡片 + 详情页倒计时每秒刷新
  - 倒计时 < 1 小时 → 显示分钟:秒数
  - 倒计时 < 1 天 → 显示小时:分钟:秒数
  - 倒计时 ≥ 1 天 → 显示天 + 时:分:秒

- [ ] **3.3** 搜索筛选优化：
  - 输入关键词后按回车或自动搜索（debounce 500ms）
  - 分类下拉选中后自动筛选
  - 筛选参数体现在 URL 上（GET 参数），支持复制分享

- [ ] **3.4** 二次确认弹窗：
  - 取消活动 → 弹窗确认"确定要取消活动吗？已报名的用户将收到通知"
  - 取消报名 → 弹窗确认"确定要取消报名吗？"
  - 拒绝报名 → 弹窗确认
  - 使用原生 `confirm()` 或自定义 Modal

- [ ] **3.5** 加载状态：
  - AJAX 加载评论时显示 loading spinner
  - 表单提交时按钮显示"提交中..."并禁用

## 4. 错误页面
