# 活动报名平台 — 需求文档

## 1. 项目概述

### 1.1 目标
用 Python 开发一个活动报名网页，让活动组织者发起活动，其他人可以在线报名参加。

### 1.2 核心流程
```
组织者创建活动 → 发布活动 → 用户浏览 & 报名 → 组织者审核 → 用户参加 → 现场签到
```

### 1.3 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | **FastAPI** | Python 异步 Web 框架 |
| 前端模板 | **Jinja2** | 服务端渲染，无需额外前端框架 |
| 数据库 | **SQLite** | 文件数据库，零配置部署 |
| CSS 框架 | **Tailwind CSS** | 现代美观的 UI 组件 |
| 异步任务 | **FastAPI BackgroundTasks** | 发送邮件等异步操作 |
| 邮件服务 | **开发阶段：控制台输出** / 上线：SMTP（QQ邮箱/SendGrid 等） | 发送真实邮件需额外配置 SMTP 服务器 |

---

## 2. 用户角色

| 角色 | 说明 | 获得方式 |
|------|------|----------|
| **普通用户** | 浏览活动、报名、取消报名 | 注册即获得 |
| **组织者** | 除普通用户权限外，还能创建和管理活动 | 注册即获得 |

> 每个用户既是普通用户（当报名别人活动时），也是组织者（当发起自己活动时）。不设管理员角色，所有功能对注册用户开放。

## 3. 功能需求

### 3.1 用户系统

| 编号 | 功能 | 说明 |
|------|------|------|
| U-01 | **注册** | 用户使用邮箱 + 密码注册账号，填写昵称 |
| U-02 | **登录** | 邮箱 + 密码登录，支持"记住我" |
| U-03 | **退出登录** | 安全退出当前会话 |
| U-04 | **个人中心** | 查看/编辑个人信息，查看报名记录，查看自己发起的活动 |

### 3.2 活动管理（组织者）

| 编号 | 功能 | 说明 |
|------|------|------|
| A-01 | **创建活动** | 填写活动信息并发布 |
| A-02 | **编辑活动** | 修改已发布的活动信息（有人报名后限制修改关键字段） |
| A-03 | **取消活动** | 取消活动，自动通知所有已报名用户 |
| A-04 | **活动列表** | 查看自己发起的所有活动，含报名人数统计 |

**活动信息字段：**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 活动标题 | 文本 | ✅ | 最多 100 字 |
| 活动描述 | 富文本 | ✅ | 支持换行、加粗等基本格式 |
| 活动分类 | 下拉选择 | ✅ | 预设分类（运动/读书/聚会/旅行/学习/其他），可搜索筛选 |
| 活动标签 | 标签 | ❌ | 自定义标签，如"户外""桌游""亲子"等，单个活动最多 5 个 |
| 封面图片 | 图片上传 | ❌ | 列表展示用，限制 2MB，存储到本地 uploads 目录 |
| 活动时间 | 日期时间 | ✅ | 开始时间 + 结束时间 |
| 活动地点 | 文本 | ✅ | 线下地址或线上链接 |
| 报名人数上限 | 数字 | ✅ | 0 表示不限制 |
| 活动费用 | 数字 | ✅ | 0 表示免费 |
| 报名截止时间 | 日期时间 | ❌ | 不填则到活动开始时截止 |

### 3.3 活动浏览 & 报名（所有用户）

| 编号 | 功能 | 说明 |
|------|------|------|
| R-01 | **活动广场** | 首页展示所有进行中的活动，支持分页 |
| R-02 | **活动搜索 & 筛选** | 按标题关键词搜索，按分类/标签/时间范围筛选 |
| R-03 | **活动详情** | 查看活动的完整信息、当前报名人数、剩余名额 |
| R-04 | **报名** | 用户点击报名，进入待审核状态 |
| R-05 | **取消报名** | 用户可以取消自己的报名（已通过审核的也可取消） |
| R-06 | **候补机制** | 报名满额后，后续报名者自动进入候补队列；有人取消/被拒后，候补第一顺位自动递补并通知 |
| R-07 | **开始倒计时** | 活动详情页和活动卡片上展示距离活动开始的倒计时（天/时/分/秒），实时更新，所有人可见 |

### 3.4 报名管理（组织者）

| 编号 | 功能 | 说明 |
|------|------|------|
| M-01 | **报名列表** | 查看某活动的所有报名者，按状态分类（待审核/已通过/已拒绝/候补） |
| M-02 | **审核报名** | 通过或拒绝报名申请，操作后自动通知报名者 |
| M-03 | **导出名单** | 导出报名成功名单为 CSV/Excel 文件 |
| M-04 | **现场签到** | 提供签到功能：组织者手动勾选签到，或生成签到二维码供参与者扫描 |

### 3.5 通知系统

| 编号 | 功能 | 说明 |
|------|------|------|
| N-01 | **站内通知** | 网页顶部通知图标 + 未读红点，展示最近通知 |
| N-02 | **邮件通知** | 以下事件发送邮件：注册成功、报名审核结果、候补转正、活动被取消 |
| N-03 | **通知列表** | 可查看历史通知记录 |

### 3.6 评论 & 留言

| 编号 | 功能 | 说明 |
|------|------|------|
| C-01 | **发表评论** | 已报名且审核通过的用户，可在活动详情页下方留言评论 |
| C-02 | **回复评论** | 组织者和其他参与者可回复评论（仅限通过审核的参与者） |
| C-03 | **删除评论** | 评论作者和组织者可删除自己的评论 |
| C-04 | **评论通知** | 活动有新评论时，通知组织者 |

### 3.7 信誉系统

| 编号 | 功能 | 说明 |
|------|------|------|
| S-01 | **失约记录** | 用户报名通过审核后未签到，记为一次失约（活动结束时统一结算） |
| S-02 | **信誉标记** | 累计失约达到 **3 次** 后，该用户再次报名任何活动时，报名信息旁会显示失约标记（如 ⚠️ 图标 + "该用户累计失约 X 次"），所有人可见 |
| S-03 | **信誉恢复** | 连续正常签到 3 次后，失约标记自动隐藏（但历史记录保留） |
| S-04 | **组织者视角** | 审核报名时，可看到报名者的失约次数，作为是否通过审核的参考 |

## 4. 页面结构

```
主页（活动广场）
├── 顶部导航栏
│   ├── Logo / 网站名称
│   ├── 搜索框
│   ├── 通知图标（含未读数）
│   └── 用户头像 → 下拉菜单（个人中心 / 我的活动 / 退出）
├── 活动卡片列表
│   └── 每张卡片：封面图、标题、时间、地点、费用、报名人数/上限
└── 底部分页

活动详情页
├── 封面图、标题、组织者
├── 时间、地点、费用、报名人数
├── 活动描述正文
├── 报名按钮 / 取消报名按钮 / "报名已满，加入候补"
└── （组织者可见）管理报名入口

创建/编辑活动页
├── 表单：标题、描述、时间、地点、人数上限、费用、截止时间、封面图
└── 发布 / 保存按钮

报名管理页（仅组织者可见）
├── 活动信息摘要
├── 报名者列表（Tab 切换：待审核 / 已通过 / 已拒绝 / 候补）
├── 审核按钮（通过 / 拒绝）
├── 导出按钮
└── 签到面板

个人中心页
├── 个人信息编辑
├── 我的报名记录
└── 我发起的活动列表
```

## 5. 非功能需求

| 编号 | 需求 |
|------|------|
| NF-01 | 密码使用 bcrypt 哈希存储，不存明文 |
| NF-02 | 网页在桌面端和移动端浏览器均可正常使用（响应式布局） |
| NF-03 | 页面加载速度 < 2 秒 |
| NF-04 | 代码通过 mypy 类型检查 + ruff 规范检查 |
| NF-05 | 所有用户输入做校验和防 XSS 处理 |

## 6. 数据库表概要

| 表名 | 主要内容 |
|------|----------|
| `users` | id, email, password_hash, nickname, avatar, no_show_count, created_at |
| `activities` | id, creator_id, title, description, cover_image, category, start_time, end_time, location, max_participants, fee, registration_deadline, status, created_at |
| `activity_tags` | id, activity_id, tag_name |
| `registrations` | id, user_id, activity_id, status(待审核/已通过/已拒绝/候补/已取消), checked_in, no_show_recorded, created_at |
| `comments` | id, user_id, activity_id, parent_id, content, created_at |
| `notifications` | id, user_id, type, title, content, is_read, created_at |

## 7. 里程碑

| 阶段 | 内容 | 验收标准 |
|------|------|----------|
| M1 | 项目骨架 + 用户系统 | 可以注册、登录、退出 |
| M2 | 活动 CRUD | 组织者可创建/编辑/查看活动，支持分类和标签 |
| M3 | 活动广场 + 报名 | 用户可浏览/搜索/筛选活动并报名/取消 |
| M4 | 审核 + 候补 + 信誉 | 组织者可审核，满额自动候补，报名时展示失约标记 |
| M5 | 通知 + 导出 + 签到 + 评论 | 邮件通知、名单导出、现场签到、活动评论 |
| M6 | UI 打磨 | 响应式布局、Tailwind 美化、交互优化 |
